In this part of documentation will be described differences from C++ API.

Qore Qt4 module is using Smoke libraries as a C++ bridge to the Qt library. When there is any feature missing it's in most cases Smoke design concept – QStrings, QMaps, etc. are missing by design. It's not a bug.

Qore is weakly typed language so there are no strict rules for variable types as in C++. For example the following code is correct and will work in the Qore Qt4 scripts:

Example 2.1. Weakly typed auto-conversions

my $var = 10;
my $lineEdit = new QLineEdit();
$lineEdit.setText( $var );

Method's (function's) input arguments are converted as much as it's possible.

QApplication

There are more constructors available for QApplication as and addition to the set of standard C++ constructors:

QApplication()

It takes the global $ARGV list. It's a simplified version of QCoreApplication( $ARGV ).

QApplication( list )

It will use the $list variable as a list with arguments.
